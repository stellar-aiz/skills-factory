# 成長ドライバー分析（売上/利益ブリッジ）PowerPoint ジェネレーター

前期→今期の売上または利益の変動を、**要因別に分解してウォーターフォールチャートで可視化**する
コンサル定番の分析スライド。「なぜ伸びたか / なぜ減ったか」を1枚で構造的に示せる。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 成長要因分析の示唆を1文で。最大70文字 |
| **チャートタイトル** | メインメッセージ直下 | 「売上成長ドライバー分析：○○ FY2023→FY2024」 |
| **ウォーターフォールチャート** | 左側（約7.8in） | 前期→各ドライバー→今期のブリッジ |
| **ドライバー解説パネル** | 右側（約4.6in） | 各ドライバーの定量影響と理由 |
| **出典** | 下部 | 情報ソースの記載 |

### ウォーターフォールチャートの構造

```
[FY2023] [+数量] [-価格] [+M&A] [-為替] [+新規] [FY2024]
  紺バー  緑バー  赤バー  緑バー  赤バー  緑バー   紺バー
```

- **始点・終点（合計）**: 紺色 #2E4A6B、0から値までの実バー
- **正の要因**: 緑 #1B7A3B、累積値からプラス分の浮きバー
- **負の要因**: 赤 #C03A3A、累積値からマイナス分の浮きバー
- **ドロップライン**: 各バーの右上端から次のバーの左上端までグレー破線でつなぐ
- **値ラベル**: 各バーの上に値を表示。正は「+150」、負は「-80」、合計は「3,000」
- **カテゴリラベル**: 各バーの下にラベル（合計はBold）

### 右パネル（ドライバー解説）

各ドライバーごとに:
- ●マーカー（緑/赤の色分け）
- ラベル（Bold）
- 値（色付き、+/-記号付き）
- 詳細説明（小さめフォント、サブテキスト色）

## 入力パターンと処理フロー

### パターンA：決算説明会資料・有報からの分析

1. **Step 1: 売上または利益の前期→今期変動を取得**
2. **Step 2: 主要ドライバーを4〜6個に分解**（数量・価格・M&A・為替・新規・コスト等）
3. **Step 3: 各ドライバーの定量影響と理由を整理**
4. **Step 4: Markdownでユーザーに提示**
5. **Step 5: 承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容確認**
2. **Step 2: 確認後、PowerPointを生成する**

### よく使う売上ドライバーの分解パターン

| カテゴリ | 例 |
|---|---|
| **既存事業の数量** | 販売数量・サービス契約数の変動 |
| **価格効果** | 平均販売単価の変動（インフレ・値下げ） |
| **製品ミックス** | 高単価製品比率の変動 |
| **M&A効果** | 買収先の通期売上連結 |
| **為替効果** | 海外売上の円換算影響 |
| **新規事業** | 新規立ち上げ事業の売上貢献 |
| **撤退事業** | 撤退・売却した事業の影響（負） |

### 利益ブリッジの場合の追加カテゴリ

| カテゴリ | 例 |
|---|---|
| **粗利率改善** | 原価低減、価格改善 |
| **販管費** | 固定費増減 |
| **減価償却** | 設備投資による償却負担増 |
| **特別損失** | リストラ費用、減損 |

## JSONデータ仕様

`{{WORK_DIR}}/growth_driver_data.json` に以下の形式で保存する：

```json
{
  "source": "出典：対象会社決算説明会資料（FY2024 Q4）、IR Q&A",
  "main_message": "売上成長+200億円のうち、既存事業の数量効果が最大貢献だが、価格競争と為替逆風が利益率を圧迫、価格戦略の見直しが急務",
  "chart_title": "売上成長ドライバー分析：対象会社 FY2023→FY2024",
  "waterfall": {
    "section_title": "売上高ブリッジ（FY2023→FY2024）",
    "unit": "億円",
    "start": {"label": "FY2023\n実績", "value": 2800},
    "end": {"label": "FY2024\n実績", "value": 3000},
    "drivers": [
      {
        "label": "既存事業\n数量",
        "value": 180,
        "detail": "国内主力製品の販売数量が前年比+6%。新型モデルAが好調で+80億円貢献"
      },
      {
        "label": "価格効果",
        "value": -80,
        "detail": "競争激化により平均販売単価が-2.8%下落"
      },
      {
        "label": "M&A\n効果",
        "value": 120,
        "detail": "2023年Q3に買収した子会社Xの通期売上が連結計上"
      }
    ]
  }
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト |
| `main_message` | string | 必須 | メインメッセージ（最大70文字） |
| `chart_title` | string | 任意 | チャートタイトル |
| `waterfall.section_title` | string | 任意 | チャート上のセクションタイトル |
| `waterfall.unit` | string | 任意 | 単位（例：「億円」「百万円」） |
| `waterfall.start.label` | string | 必須 | 始点ラベル（例：「FY2023\n実績」、改行可） |
| `waterfall.start.value` | number | 必須 | 始点の値 |
| `waterfall.end.label` | string | 必須 | 終点ラベル |
| `waterfall.end.value` | number | 必須 | 終点の値 |
| `waterfall.drivers` | array | 必須 | ドライバー配列（3〜6個推奨） |
| `waterfall.drivers[].label` | string | 必須 | ドライバー名（チャート用、改行\n使用可） |
| `waterfall.drivers[].value` | number | 必須 | 定量影響（正/負） |
| `waterfall.drivers[].detail` | string | 任意 | 解説（右パネルに表示） |

### ラベルの改行（`\n`）について

- チャート（左）の `label` は2行表示で見やすくするため `\n` 使用OK
- 右パネルでは自動的に空白に置換されて1行表示

## 注意事項

- **数値整合性**: start.value + Σ(drivers.value) = end.value となるよう必ず確認
- **ドライバー数**: 3〜6個が読みやすい。多すぎると棒が細くなる
- **売上 vs 利益**: 同じ枠組みで両方使えるが、混在させない（1スライド1指標）
- **複数年の場合**: 1スライドは1組（前期→今期）。複数組必要なら別スキル検討

## 企業調査レポートでの使用例

### 配置位置（推奨）

戦略的示唆セクションの一部として配置。
バリューチェーン分析と並行で配置すると、「現状の構造」+「変動の構造」の両面が示せる。

### 価値の出し方

「成長したからOK」ではなく、**「成長の質」を可視化**できる。
例: 売上+200億円でも「価格は-80億円下落、数量と新規で稼いだ」と分かれば、
**価格戦略の見直しが必要**という示唆につながる。

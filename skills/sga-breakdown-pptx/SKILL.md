# 対売上販管費比率と販管費内構成比の推移 PowerPoint ジェネレーター

BDD（ビジネスデュー・ディリジェンス）や財務分析において、販管費の各科目を
積み上げ棒グラフ＋折れ線（対売上高販管費比率）で1枚のPowerPointスライドに整理するスキル。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「対売上販管費比率と、販管費内構成比の推移」等 |
| **単位ラベル** | チャート左上 | 「（単位：百万円、%）」等 |
| **チャートエリア** | 左側（約65%） | 積み上げ棒グラフ＋折れ線グラフ＋トレンド矢印 |
| **凡例** | 右側（約35%） | 2列レイアウトで科目名＋色マーカー |
| **出典** | 左下 | 情報ソースの記載 |

### チャートエリアの構成

- **積み上げ棒グラフ**: 販管費科目数（最大30科目）の積み上げ
- **データラベル**: 各セグメント内に「金額\n(構成比%)」を白文字で表示（閾値以下は非表示）
- **合計ラベル**: 棒の上にテキストボックスで合計値を表示
- **折れ線グラフ**: 対売上高販管費比率（第2軸、非表示）
- **トレンド矢印**: 期間間の比率変化を矢印＋テキストで可視化

### 右側凡例

- 折れ線の凡例（最上部）
- 販管費科目の凡例を2列レイアウト
- 値がゼロの科目は自動的に凡例から除外

## 入力パターンと処理フロー

### パターンA：Excelファイルがアップロードされた場合（メインフロー）

**いきなりPowerPointを作成しない。必ず以下のステップを踏む。**

#### Step 1: Excelファイルの読み取り

Excelファイルを `parse_sga_excel.py` で読み取る。
期ごとにシートが分かれている想定（シート名が期ラベルになる）。

**Claudeが確認すべき項目:**
- 各シートの科目名と金額が正しく読み取れたか
- 売上高データ（販管費比率計算に必要）がExcel内にあるか、別途ユーザーから取得が必要か

#### Step 2: データ整理結果の確認

抽出したデータをMarkdownテーブルでユーザーに提示し、確認・修正を求める。

**提示フォーマット:**

```markdown
## 販管費構成比 整理結果

### 各期のデータ

| 科目 | 21/6期 | 22/6期 | 23/6期 |
|------|--------|--------|--------|
| 役員報酬 | 2 | 2 | 2 |
| 給料手当 | 8 | 8 | 8 |
| ... | ... | ... | ... |
| **合計** | **43** | **42** | **41** |

### 対売上高販管費比率
| 期 | 売上高 | 販管費合計 | 比率 |
|----|--------|----------|------|
| 21/6期 | 370 | 43 | 11.6% |
| 22/6期 | 424 | 42 | 9.9% |
| 23/6期 | 661 | 41 | 6.2% |
```

**確認時の質問（ask_user_inputで提示）:**
```
質問: データの内容でよろしいですか？
選択肢:
  - このままPPT生成
  - データを修正したい
  - メインメッセージを変更したい
```

#### Step 3: ユーザー承認後、PowerPointを生成

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、科目・金額・売上高が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成**

## Excelフォーマット仕様

### シート構成

- **シート名**: 期ラベル（例: "21/6期", "22/6期", "23/6期"）
- 各シートにヘッダー行＋データ行

### ヘッダー自動判定

以下のキーワードを含む列を自動検出:
- **科目名列**: 科目、費目、項目、勘定、名称、内訳
- **金額列**: 金額、額、合計、実績、千円、百万円

### 除外ルール

- 「合計」「計」「Total」「販管費計」を含む行は自動スキップ

## JSONデータ仕様

`{{WORK_DIR}}/sga_breakdown_data.json` に以下の形式で保存する：

```json
{
  "main_message": "販管費比率は11.6%→6.2%に低下、コスト管理の成果を評価すべき",
  "chart_title": "対売上販管費比率と、販管費内構成比の推移",
  "source": "対象会社提供資料",
  "unit_label": "（単位：百万円、%）",
  "label_threshold_pct": 5.0,
  "categories": [
    {"name": "役員報酬", "color": "#2E7D32"},
    {"name": "給料手当", "color": "#4472C4"},
    {"name": "地代・家賃", "color": "#4E79A7"}
  ],
  "line": {
    "series_name": "対売上高販管費比率",
    "color": "#666666"
  },
  "periods": [
    {
      "label": "21/6期",
      "values": [2, 8, 12],
      "total": 43,
      "line_value": 11.6
    }
  ],
  "trend_arrow": true
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `main_message` | string | 必須 | メインメッセージ。最大70文字、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「対売上販管費比率と、販管費内構成比の推移」 |
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `unit_label` | string | 任意 | 単位表記（左上に表示） |
| `label_threshold_pct` | number | 任意 | データラベル表示閾値（%）。デフォルト5.0。構成比がこの値未満のセグメントはラベル非表示 |
| `categories` | array | 必須 | 販管費科目の定義（積み上げ順） |
| `categories[].name` | string | 必須 | 科目名 |
| `categories[].color` | string | 必須 | 色コード（例: "#4E79A7"） |
| `line` | object | 任意 | 折れ線グラフの定義。省略可 |
| `line.series_name` | string | 必須 | 系列名（例: "対売上高販管費比率"） |
| `line.color` | string | 任意 | 色コード。デフォルト "#666666" |
| `periods` | array | 必須 | 期別データの配列 |
| `periods[].label` | string | 必須 | 期ラベル（例: "21/6期"） |
| `periods[].values` | array | 必須 | 各科目の金額（categoriesの順序に対応） |
| `periods[].total` | number | 必須 | 販管費合計 |
| `periods[].line_value` | number | 任意 | 折れ線の値（対売上高販管費比率%） |
| `trend_arrow` | boolean | 任意 | トレンド矢印の表示。デフォルト true |

## スクリプト実行コマンド

### Excel → JSON変換

```bash
pip install python-pptx openpyxl -q --break-system-packages

python <SKILL_DIR>/scripts/parse_sga_excel.py \
  --excel {{INPUT_DIR}}/sga_data.xlsx \
  --sales '{"21/6期": 370, "22/6期": 424, "23/6期": 661}' \
  --output {{WORK_DIR}}/sga_breakdown_data.json
```

### JSON → PPTX生成

```bash
python <SKILL_DIR>/scripts/fill_sga_breakdown.py \
  --brand {stellar_aiz|roleup} \
  --data {{WORK_DIR}}/sga_breakdown_data.json \
  --output {{OUTPUT_DIR}}/SGA_Breakdown_output.pptx
```

`--brand` 省略時は `stellar_aiz`(16:9, Meiryo UI, V1 配色)。
`--brand roleup` 指定時は A4 横テンプレ + Yu Gothic UI 10pt + 茶系 line color (theme.accent_op_margin_line) + Source 3 placeholder 6pt。
`--template` 明示指定で brand_resolver の解決を上書き可能(通常不要)。

※ `<SKILL_DIR>` は実際のスキルインストールパスに置き換えること。

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/SGA_Breakdown_output.pptx
```

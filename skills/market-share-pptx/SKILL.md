# 市場シェア分析 PowerPoint ジェネレーター

対象市場における主要プレイヤーのシェア構造と推移を1枚のPowerPointスライドに整理するスキル。
BDD（ビジネスデュー・ディリジェンス）・競合分析・市場参入検討・投資判断において、
「対象市場の寡占度」「対象会社のランキング位置」「シェア変動のダイナミクス」を可視化する。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大65文字（hard-fail）。シェア分析からの示唆を「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「市場シェア分析：○○市場」 |
| **ドーナツチャート** | 左側 | 最新年の市場シェア円グラフ＋凡例＋市場規模ラベル |
| **シェア推移テーブル** | 右側 | ランク・企業名・各年シェア・YoY変化の一覧表 |
| **出典** | 左下 | 情報ソースの記載 |

### 左側: ドーナツチャート

- ネイティブ PowerPoint チャート（XL_CHART_TYPE.DOUGHNUT）
- **ホールサイズ55%**でスタイリッシュに表示
- 各スライスは指定色または自動割り当て色で表示
- スライスの境界は白線(1.5pt)で区切り
- データラベル: パーセント値を中央に白文字で表示（10pt Bold）
- 凡例: 右側に縦並び配置（企業名表示）
- 上部に市場規模ラベル（例：「市場規模: 29.6兆円」）

### 右側: シェア推移テーブル

**列構成:**
- `#` (ランク): 最新年のシェアで自動ソート・採番。「その他」は「—」表記
- `企業名`: Bold + セグメント色の「■」マーカー（ドーナツチャートと色連動）
- `年1, 年2, ..., 年N`: 各年のシェア（小数点以下1桁、%付き）
- `YoY変化`: 最新年−前年 = 変化量。色分け：
  - **↑ +X.Xpt** (緑 #1B7A3B): 上昇 (+0.1pt超)
  - **↓ -X.Xpt** (赤 #C03A3A): 下降 (-0.1pt超)
  - **→ ±X.Xpt** (グレー #666666): 変化なし (±0.1pt以内)

**行の特徴:**
- **対象会社行はイエロー背景 (#FFF4C2)** でハイライト
- 通常行はゼブラストライピング（奇数行=薄灰、偶数行=白）
- 「その他」は最終行に固定配置

### 年度範囲のルール

- **End year**: 実行年の前年
- **年度数**: 2〜5年を推奨（3年が最もバランス良い）
- **ランク**: 最新年のシェアで自動ソート

## 入力パターンと処理フロー

### パターンA：業界統計データが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: 業界・各社シェアを抽出・整理する**
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、項目が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

### シェア情報のデータソース

| 業界 | 主な情報源 |
|---|---|
| 消費財・小売 | 矢野経済研究所、富士経済、インテージ |
| 自動車・機械 | MarkLines、JAMA、業界団体 |
| IT・ソフトウェア | IDC Japan、Gartner、MM総研 |
| 金融・保険 | 金融庁統計、生命保険協会、各社開示資料 |
| 医薬品 | IQVIA、日経メディカル、各社IR |
| 食品・飲料 | 食品産業新聞、帝国データバンク |
| 建設・不動産 | 建設経済研究所、不動産経済研究所 |

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト |
| `main_message` | string | 必須 | メインメッセージ（最大65文字（hard-fail）） |
| `chart_title` | string | 任意 | チャートタイトル |
| `target_company` | string | 任意 | 対象会社名（テーブル上でイエローハイライト） |
| `years` | array | 必須 | 年度ラベル配列（2〜5年を推奨） |
| `players` | array | 必須 | プレイヤー定義配列 |
| `players[].name` | string | 必須 | 企業名（「その他」は特別扱い：ランク「—」、最下行に固定） |
| `players[].color` | string | 任意 | 色（#RRGGBB）。未指定時は自動割り当て |
| `players[].shares` | array | 必須 | 各年のシェア値（%）。years と同じ長さ |
| `chart_year_index` | int | 任意 | ドーナツチャートで表示する年のインデックス。デフォルト=最新年 |
| `left_panel.section_title` | string | 任意 | 左側セクションタイトル |
| `left_panel.total_size_label` | string | 任意 | 市場規模表記（チャート上部） |
| `right_panel.section_title` | string | 任意 | 右側セクションタイトル |
| `right_panel.show_ranking` | bool | 任意 | ランク列の表示有無。デフォルト=true |
| `right_panel.show_yoy_change` | bool | 任意 | YoY変化列の表示有無。デフォルト=true |

### 「その他」の扱い

- `players` 配列内に `{"name": "その他", ...}` として含める
- 色は自動的に灰色（#BAB0AC）
- テーブル上ではランク欄が「—」となり、最下行に固定配置
- ドーナツチャートでは通常のスライスとして表示

### プレイヤー数

- **推奨**: 上位5〜8社 + 「その他」
- 少なすぎる（3社未満）と寡占感が伝わらない
- 多すぎる（10社超）とドーナツチャートが読みにくくなる

## スクリプト実行コマンド

```bash
pip install python-pptx -q --break-system-packages

python <SKILL_DIR>/scripts/fill_market_share.py \
  --data {{WORK_DIR}}/market_share_data.json \
  --brand stellar_aiz \
  --output {{OUTPUT_DIR}}/MarketShare_output.pptx
```

`--brand` を `roleup` に切り替えると Roleup ブランド (A4 横、Yu Gothic UI、褐色アクセント) で生成。
`--template` は省略可（brand から `assets/<brand>/market-share-template.pptx` を自動解決）。
明示指定したい場合は `--template <SKILL_DIR>/assets/<brand>/market-share-template.pptx` を渡す。

## 注意事項

- **シェアの合計**: 全プレイヤーの合計が100%になるよう整合を取る（「その他」で調整）
- **YoY変化の閾値**: ±0.1pt以内は「→」（変化なし）、±0.1pt超は↑/↓と表記
- **対象会社の特定**: `target_company` フィールドに正確な企業名を指定。`players[].name` と完全一致させる
- **小さいシェアの表示**: 1%未満のプレイヤーはドーナツチャートでラベルが表示されないことがあるが、テーブルには反映される
- **情報源の信頼性**: 業界統計・一次資料を優先。推計値・第三者レポートは出典に明記する

## 競合分析デッキでの使用例

`competitor-analyst-agent` オーケストレータースキルから呼び出される位置づけ:

```
1. 対象会社プロファイル (customer-profile-pptx)
2. 対象会社 事業ポートフォリオ (business-portfolio-pptx)
3. 対象会社 SWOT (swot-pptx)
4. 市場環境 - 市場規模推移 (market-environment-pptx)
5. 市場環境 - 市場シェア分析 ← 本スキル
6. 競合比較サマリー (competitor-summary-pptx)
7. 各競合プロファイル (customer-profile-pptx × N)
```

4枚目（市場規模の推移）と5枚目（市場シェアの構造）を並べることで、
「市場の大きさとダイナミクス」＋「誰が主要プレイヤーか」の両面から市場環境を俯瞰できる。

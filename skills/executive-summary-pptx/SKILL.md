# エグゼクティブサマリー PowerPoint ジェネレーター

企業調査レポート（BDD・競合分析・M&Aターゲット評価・新規参入調査等）の冒頭に配置する
**1枚でストーリーが追えるサマリースライド**を生成するスキル。

戦略コンサルティングファームや投資銀行が作成する調査レポートでは、
**冒頭のエグゼクティブサマリーが最重要**（忙しい経営層はここしか読まないことも）。
3〜5個のKey Findingsを構造化して、調査の核心を1枚で伝える。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 調査全体の結論を1文で。最大65文字（hard-fail）、「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「エグゼクティブサマリー：○○調査」 |
| **Key Findings** | 中央 | 3〜5個のFindingを縦積みで配置 |
| **出典** | 下部 | 情報ソースの記載 |

### 各Findingの構成

```
[01]▍対象会社  見出し (Bold 14pt)
    詳細テキスト（2-3行、11pt、サブテキスト色）
```

- **番号バッジ**: 円形、カテゴリ色、白文字で番号（01〜05）
- **縦バー**: 左側の縦ライン（カテゴリ色）、Findingの範囲を示す
- **カテゴリラベル**: 小さなタグ風（「▍カテゴリ」、Bold 10pt、カテゴリ色）
- **見出し**: 14pt Bold、黒、1行の強いメッセージ
- **詳細**: 11pt Regular、サブテキスト色、2-3行の補足説明

### カテゴリと色マッピング

| カテゴリ | 色 | 用途例 |
|---|---|---|
| **対象会社** / target / company | 紺 #2E4A6B | 対象会社の事業・財務・強み |
| **マクロ環境** / macro / pest | 紫 #7B4FB0 | PEST・外部環境 |
| **市場** / market | 青 #2E6FBF | 市場規模・成長率・シェア |
| **競合** / competitor | オレンジ #DA7A2D | 競合動向・ポジショニング |
| **財務** / financial | 緑 #3D8F5A | 財務ベンチマーク・収益性 |
| **リスク** / risk | 赤 #B83A3A | 懸念事項・脅威 |
| **機会** / opportunity | 濃緑 #1B7A3B | 成長機会・追い風 |
| **示唆** / implication | グレー #555555 | 含意・So What |
| **結論** / conclusion | 濃グレー #333333 | 推奨アクション・最終結論 |

未知のカテゴリはデフォルト色（紺 #4E79A7）。
`color` フィールドで明示的にHEX色を指定することも可能。

## 入力パターンと処理フロー

### パターンA：調査レポート全体の内容から要約を作る場合

**いきなりPowerPointを作成しない**

1. **Step 1: 調査データから3〜5個のKey Findingsを抽出**
   - 通常は「対象会社 / マクロ・市場 / 競合 / 財務 / 結論」の5つに構成
   - 3つの場合は「対象会社 / 市場・競合 / 結論」
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：Key Findingsが整理済みで入力された場合

1. **Step 1: 内容を確認**し、3〜5個のFindingが揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

### Findings構成の推奨パターン

**5 Findings パターン（拡張版レポート向け）:**
1. 対象会社の現状（事業規模・ポジション・強み）
2. マクロ環境（PEST・外部環境の追い風/逆風）
3. 市場（市場規模・成長率・シェア動向）
4. 競合（競合の相対ポジション・差別化）
5. 結論（推奨アクション）

**3 Findings パターン（簡潔版）:**
1. 対象会社の現状
2. 市場・競合環境
3. 結論・推奨アクション

## JSONデータ仕様

`{{WORK_DIR}}/executive_summary_data.json` に以下の形式で保存する：

```json
{
  "source": "出典：Web調査（対象会社IR・業界レポート・各社HP、2025年度までのデータ）",
  "main_message": "対象会社は国内シェア2位の安定基盤を持つが、EV化の波に乗り遅れており、海外展開を含む事業ポートフォリオの再編が急務",
  "chart_title": "エグゼクティブサマリー：対象会社競合分析",
  "findings": [
    {
      "category": "対象会社",
      "heading": "国内市場では強固な地位を維持（売上シェア約15%、業界2位）",
      "detail": "自動車事業が売上の88%を占めるが、金融・その他事業が利益率を底上げ。..."
    },
    {
      "category": "マクロ環境",
      "heading": "政策・技術は追い風、経済環境は逆風",
      "detail": "..."
    },
    {
      "category": "市場",
      "heading": "市場は拡大基調だが、新興勢のシェア急伸",
      "detail": "..."
    },
    {
      "category": "競合",
      "heading": "テスラ・BYDが成長率で大幅に先行",
      "detail": "..."
    },
    {
      "category": "結論",
      "heading": "推奨アクション：EV・海外投資の加速",
      "detail": "..."
    }
  ]
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト |
| `main_message` | string | 必須 | 調査全体の結論メッセージ（最大65文字（hard-fail）） |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「エグゼクティブサマリー」 |
| `findings` | array | 必須 | Finding配列（3〜5個、最大6個） |
| `findings[].category` | string | 任意 | カテゴリ名（色自動割り当て）。上記マッピング表参照 |
| `findings[].heading` | string | 必須 | 見出し（Bold 14pt）。1行で強いメッセージを |
| `findings[].detail` | string | 必須 | 詳細テキスト（2-3行、11pt）。補足説明・定量データ |
| `findings[].color` | string | 任意 | カテゴリ色の明示指定（#RRGGBB）。未指定時はcategoryから自動取得 |

## スクリプト実行コマンド

```bash
pip install python-pptx -q --break-system-packages

python <SKILL_DIR>/scripts/fill_executive_summary.py \
  --brand stellar_aiz \
  --data {{WORK_DIR}}/executive_summary_data.json \
  --output {{OUTPUT_DIR}}/ExecutiveSummary_output.pptx
```

`--brand` は `stellar_aiz`（既定）または `roleup` を指定。`--template` は省略可（指定しない場合は brand 別に解決される）。
- `stellar_aiz`: `assets/stellar_aiz/executive-summary-template.pptx`(13.33×7.50 in、Meiryo UI、青系アクセント)
- `roleup`: `assets/roleup/executive-summary-template.pptx`(11.69×8.27 in、Yu Gothic UI、褐色アクセント、本文 12pt、出所必須)

### レイアウト定数

| 要素 | 値 |
|---|---|
| グリッド開始Y | 1.55in |

## 品質チェックリスト

- [ ] メインメッセージが最大65文字（hard-fail）以内で調査全体を俯瞰しているか
- [ ] Findingsが3〜5個の適切な範囲か
- [ ] 各Findingの番号バッジ・縦バーの色が一貫しているか
- [ ] カテゴリラベルが見出しに重ならず読みやすいか
- [ ] 見出しが1行に収まり、強いメッセージになっているか
- [ ] 詳細テキストが2-3行で具体的な数字・事実を含んでいるか
- [ ] 最後のFindingが「結論」または「推奨アクション」になっているか
- [ ] 出典が適切に記載されているか

## 注意事項

- **Findingsの順序**: 一般的には「対象会社 → マクロ → 市場 → 競合 → 結論」の外→内→結論の流れが分かりやすい
- **見出しの書き方**:
  - **Good**: 「国内シェア2位で安定基盤だが、EV化で海外勢に出遅れ」（事実＋含意）
  - **NG**: 「国内市場について」（タイトルのみで内容がない）
- **詳細テキストの書き方**: 具体的な数値・パーセンテージ・年度を含める。抽象的な記述は避ける
- **カテゴリの使い分け**:
  - 「リスク」「機会」は特定の観点を強調したい場合に使用
  - 「結論」は最後のFindingで必ず使うことを推奨（オレンジや赤で目立たせると効果的）
- **色の統一感**: 他のスライド（例: PEST分析の色と）と合わせることで、デッキ全体の一貫性が出る

## 企業調査レポートでの使用例

### 例1: 標準的な競合分析レポート（10枚構成）

```
1. エグゼクティブサマリー ← 本スキル
2. 対象会社プロファイル
3. 対象会社 事業ポートフォリオ
4. 対象会社 SWOT
5. マクロ環境 PEST分析
6. 市場環境 - 市場規模推移
7. 市場環境 - 市場シェア分析
8. 市場環境 - ポジショニングマップ
9. 競合比較サマリー
10. 財務ベンチマーク
```

### 例2: M&Aターゲット評価レポート

```
1. エグゼクティブサマリー ← 本スキル
2. ターゲット企業プロファイル
3. 事業ポートフォリオ
4. 財務ベンチマーク
5. バリュエーション分析
6. ディール構造・リスク
```

### 例3: 新規参入調査レポート

```
1. エグゼクティブサマリー ← 本スキル
2. マクロ環境 PEST分析
3. 市場規模・成長性
4. 市場シェア・寡占度
5. 既存プレイヤーのポジショニング
6. 参入障壁・成功要因
7. 推奨参入戦略
```

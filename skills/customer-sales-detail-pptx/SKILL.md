# 主要販売先詳細テーブル PowerPoint ジェネレーター

BDD（ビジネスデュー・ディリジェンス）において、対象会社の主要販売先ごとの売上実績に加え、
各顧客の企業プロファイル（事業内容・本社所在地・上場区分・直近期業績）を
1枚のテーブルスライドにまとめるスキル。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「主要販売先からの完成工事売上高と割合(XXXX/X期-X月累計)」等 |
| **詳細テーブル** | コンテンツエリア | 上位10社＋その他の1テーブル（10列） |
| **出典** | 左下 | 情報ソースの記載 |

### テーブルの列構成

| 列 | 内容 | 備考 |
|---|---|---|
| # | 順位（1〜10） | その他は空欄 |
| 企業名 | 顧客企業名 | 左寄せ |
| 売上高(千円) | 対象会社からの売上高 | カンマ区切り、右寄せ |
| 割合 | 全体に占める割合（%） | 小数点1桁、右寄せ |
| 事業内容 | 顧客企業の事業概要 | 左寄せ、折り返し |
| 本社所在地 | 顧客企業の本社所在地 | 左寄せ |
| 上場 | 上場区分（東証プライム/未上場等） | 左寄せ |
| 直近期売上高(千円) | 顧客企業の直近期売上高 | カンマ区切り、右寄せ。NA対応 |
| 利益(千円) | 顧客企業の直近期利益 | カンマ区切り、右寄せ。NA対応。負値対応 |
| 利益率 | 顧客企業の直近期利益率 | 小数点1桁、右寄せ。NA対応。負値対応 |

---

## 入力パターンと処理フロー

### パターンA：Excelファイル＋TSR情報がアップロードされた場合

**いきなりPowerPointを作成しない**

1. **Step 1: Excelを読み取る**
   - 企業名と売上高を読み取る
   - 売上高の合計から各社の割合を算出
   - 売上高降順でランク付け、上位10社＋その他を集計

2. **Step 2: TSR情報を統合する**
   - ユーザーが提供したTSR情報（事業内容・本社所在地・上場・直近期売上高・利益・利益率）を
     各企業に紐づける
   - TSR情報がない企業はNA表示

3. **Step 3: Markdownでユーザーに提示**し、確認・修正を求める

4. **Step 4: ユーザーの承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、項目が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: Excelの読み取り

### 想定Excelフォーマット

**各シートの列構成**:

| 列 | 内容 |
|---|---|
| 企業名（A列等） | 顧客企業名 |
| 売上高（B列等） | 売上高（数値） |

※ 列名が「企業名」「顧客名」「得意先」「取引先」等の場合は企業名列と判定
※ 列名が「売上高」「売上」「金額」「完成工事高」等の場合は売上高列と判定

### 集計ルール

1. 全顧客の売上高を合計 → 全体売上高
2. 売上高降順でソート → 上位10社を抽出
3. 11位以下を「その他」として合算
4. 各社の割合 = 各社売上高 ÷ 全体売上高 × 100（小数点1桁）

---

## Step 2: TSR情報の統合

ユーザーが提供するTSR（東京商工リサーチ）情報を各企業に紐づける。

### TSR情報の項目

| 項目 | 説明 | 例 |
|---|---|---|
| 事業内容 | 企業の主な事業 | 「発送代行業、事務代行業、アウトソーシング、充電」 |
| 本社所在地 | 本社の所在地 | 「瀬戸内市長船町長船」 |
| 上場 | 上場区分 | 「未上場」「東証プライム」「Bプライム」 |
| 直近期売上高 | 直近期の売上高（千円） | 19885873（数値またはNA） |
| 利益 | 直近期の利益（千円） | 683905（数値またはNA。負値可） |
| 利益率 | 直近期の利益率（%） | 3.4（数値またはNA。負値可） |

### NA値の扱い

- TSR情報が取得できない企業は「NA」と表示
- 「その他」行のTSR列は全て空欄

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `main_message` | string | 必須 | メインメッセージ。最大70文字、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「主要販売先からの完成工事売上高と割合」 |
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `unit` | string | 任意 | 売上高の単位。デフォルト「千円」 |
| `customers` | array | 必須 | 顧客データの配列（上位10社＋その他） |
| `customers[].rank` | int | 必須 | 順位（1〜10）。その他は -1 |
| `customers[].name` | string | 必須 | 企業名 |
| `customers[].revenue` | number | 必須 | 対象会社からの売上高 |
| `customers[].share` | number | 必須 | 割合（%） |
| `customers[].business` | string/null | 任意 | 事業内容。nullなら空欄 |
| `customers[].headquarters` | string/null | 任意 | 本社所在地。nullなら空欄 |
| `customers[].listing` | string/null | 任意 | 上場区分。nullなら空欄 |
| `customers[].latest_revenue` | number/null | 任意 | 直近期売上高。nullなら「NA」表示 |
| `customers[].profit` | number/null | 任意 | 直近期利益。nullなら「NA」表示 |
| `customers[].profit_margin` | number/null | 任意 | 直近期利益率(%)。nullなら「NA」表示 |

### スクリプト実行コマンド

```bash
apt-get install -y -qq fonts-noto-cjk 2>/dev/null
pip install python-pptx playwright openpyxl -q --break-system-packages
playwright install chromium 2>/dev/null

python <SKILL_DIR>/scripts/fill_customer_sales_detail.py \
  --data {{WORK_DIR}}/customer_sales_detail_data.json \
  --output {{OUTPUT_DIR}}/CustomerSalesDetail_output.pptx \
  --brand stellar_aiz
```

`--brand roleup` を渡すと A4 横 + Yu Gothic UI + 茶系のテンプレ・配色で生成される。
`--template` は任意 (省略時は brand 別 curated テンプレを自動解決)。

※ `<SKILL_DIR>` は実際のスキルインストールパスに置き換えること。

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/CustomerSalesDetail_output.pptx
```

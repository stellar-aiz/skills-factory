# SWOT分析 PowerPoint ジェネレーター

対象会社・競合・事業などのSWOT分析を1枚のPowerPointスライドに整理する汎用スキル。
BDD（ビジネスデュー・ディリジェンス）・競合分析・経営戦略立案・投資検討など、幅広い文脈で使用可能。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。SWOTから導かれる示唆を「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「SWOT分析：○○社」 |
| **強み (Strengths)** | 左上 | 紺背景のヘッダー + 薄い紺の本体、ブレット項目 |
| **弱み (Weaknesses)** | 右上 | オレンジ背景のヘッダー + 薄いオレンジの本体、ブレット項目 |
| **機会 (Opportunities)** | 左下 | 緑背景のヘッダー + 薄い緑の本体、ブレット項目 |
| **脅威 (Threats)** | 右下 | 赤背景のヘッダー + 薄い赤の本体、ブレット項目 |
| **出典** | 左下 | 情報ソースの記載 |

### マトリクスレイアウトの論理

```
           [内部要因 (Internal)]      [内部要因 (Internal)]
[プラス]    強み (Strengths)          弱み (Weaknesses)
           [外部要因 (External)]      [外部要因 (External)]
[マイナス]  機会 (Opportunities)     脅威 (Threats)
```

- 日本のビジネスドキュメントで最も一般的な「上段: 内部要因、下段: 外部要因」レイアウト
- 左列: プラス要因、右列: マイナス要因 は伝統的配置

### 各象限の構成

- **ヘッダーバー** (高さ0.55in):
  - 象限色（紺・オレンジ・緑・赤）の背景
  - 日本語ラベル（大、16pt Bold）+ 英語ラベル（小、11pt）を白文字で表示
- **ボディ** (残り):
  - 薄い象限色の背景
  - ブレット項目を「• 〜」形式で縦に列挙
  - ブレット色は象限のメインカラーと一致
  - 文字色は #333333（黒系）、フォント Meiryo UI 12pt

## 入力パターンと処理フロー

### パターンA：企業情報（HP・IR・業界レポート等）が入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: 情報からSWOT 4象限の項目を抽出・整理する**
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：SWOTが整理済みで入力された場合

1. **Step 1: 内容を確認**し、各象限が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

### SWOT抽出のガイドライン

Webから情報を集めてSWOTを構築する場合、各象限で以下を探す:

| 象限 | 視点 | 情報源の例 |
|---|---|---|
| **強み (S)** | 他社より優れている内部要因 | 市場シェア、技術力、ブランド、財務、人材、オペレーション、顧客基盤 |
| **弱み (W)** | 他社より劣っている内部要因 | コスト構造、組織体制、技術的遅れ、地域カバレッジ、人材不足 |
| **機会 (O)** | 追い風になる外部要因 | 市場拡大、規制緩和、技術革新、顧客行動変化、競合の弱体化 |
| **脅威 (T)** | 逆風になる外部要因 | 新規参入、規制強化、技術代替、市場縮小、為替・原材料リスク |

**項目数の目安**: 各象限 3〜6個（5個前後が最もバランス良い）

## JSONデータ仕様

`{{WORK_DIR}}/swot_data.json` に以下の形式で保存する：

```json
{
  "source": "出典：対象会社有価証券報告書・中期経営計画（2024年）、業界レポート",
  "main_message": "当社は強固な国内シェアと技術基盤を持つ一方、海外展開の遅れが成長機会を逃すリスクとなっており、EV化を契機とした海外投資を加速すべき",
  "chart_title": "SWOT分析：トヨタ自動車株式会社",
  "swot": {
    "strengths": {
      "items": [
        "国内自動車販売シェア約45%（圧倒的No.1ポジション）",
        "HV技術で20年以上の蓄積、累計販売2,500万台超",
        "全国約4,600店舗の強固なディーラー網"
      ]
    },
    "weaknesses": {
      "items": [
        "EV販売比率が1%未満、欧米中勢に対して出遅れ",
        "ソフトウェア人材の不足、OTA更新対応の遅れ"
      ]
    },
    "opportunities": {
      "items": [
        "世界EV市場の急拡大（2030年に新車の30%以上がEV予測）",
        "東南アジア・インド市場の中間層拡大に伴うモータリゼーション"
      ]
    },
    "threats": {
      "items": [
        "テスラ・BYD等の新興EVメーカーの急速な台頭",
        "中国政府によるEV優遇策・国産優先による市場閉鎖"
      ]
    }
  }
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `main_message` | string | 必須 | メインメッセージ。最大70文字、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「SWOT分析」 |
| `swot.strengths.items` | array | 必須 | 強みのブレット項目配列（3〜6個推奨） |
| `swot.weaknesses.items` | array | 必須 | 弱みのブレット項目配列（3〜6個推奨） |
| `swot.opportunities.items` | array | 必須 | 機会のブレット項目配列（3〜6個推奨） |
| `swot.threats.items` | array | 必須 | 脅威のブレット項目配列（3〜6個推奨） |
| `swot.*.label_jp` | string | 任意 | 日本語ラベルの上書き（例：「内部強み」） |
| `swot.*.label_en` | string | 任意 | 英語ラベルの上書き |

## 注意事項

- **項目の長さ**: 各項目は50文字以内を推奨。長すぎると折り返しで読みにくくなる
- **項目数**: 5個/象限がベストバランス。6個超えると文字が詰まって読みにくい
- **対象**: 対象会社だけでなく、競合・買収候補・事業部単位などどんな対象にも適用可能
- **客観性**: 情報源はWeb上の一次情報（公式HP・IR・業界レポート）を優先。推測や主観を避け、必ず根拠を持つ項目に絞る
- **メインメッセージ**: 4象限を俯瞰して得られる戦略的示唆を1文で表現する。単なる事実羅列ではなく「So What?」を含める

## 競合分析デッキでの使用例

`competitor-analyst-agent` オーケストレータースキルから呼び出される想定の位置づけ:

```
1. 対象会社プロファイル (customer-profile-pptx)
2. 対象会社 事業ポートフォリオ (business-portfolio-pptx)
3. 対象会社 SWOT ← 本スキル
4. 市場環境 (market-environment-pptx)
5. 競合比較サマリー (competitor-summary-pptx)
6. 各競合プロファイル (customer-profile-pptx × N)
   [7. 各競合 SWOT (swot-pptx × N) 追加オプション]
```

対象会社のSWOT分析は「なぜこの会社が投資・買収対象として魅力的か / 何がリスクか」のエグゼクティブサマリーとして機能する。
各競合のSWOTも追加することで、競合ごとの戦略ポジションが比較しやすくなる。

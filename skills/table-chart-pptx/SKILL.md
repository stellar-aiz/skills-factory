# Table Chart PowerPoint ジェネレーター

データをテーブル形式で比較・一覧化し、意味合い（Implications）を添えた戦略コンサルタント定番のスライドを生成するスキル。

---

## テーブルチャートとは

左側にデータテーブル、右側に意味合いボックスを配置したスライドフォーマット。定量データの比較・分析結果を一覧化し、そこから導かれるビジネス上の示唆を右側にまとめる。

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | テーブル全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。テーブルのテーマや文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **Table Label** | テーブルセクションの見出し。テーブルの内容を一言で表す（例: 「主要競合の比較」「拠点別KPI一覧」） | 何のテーブルかが一目で分かること |
| **テーブル** | ヘッダー行 + データ行で構成。行数・列数はコンテンツに応じて動的に決定 | 定量データで構成し、客観的に比較可能にする |
| **意味合い（Implications）** | テーブルのデータから導かれるビジネス上の示唆。1〜5点 | テーブルの数値・事実を根拠にした具体的な示唆にする |

### 記述ルール

- **Main Message**: 最大70文字。テーブルと意味合いを一言で総括し、必ず「〜すべき」で終える。ユーザーが指定した場合はそのまま使用、指定がない場合はドラフトして確認を取る。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。ユーザーが指定しない場合はドラフトして確認を取る。
- **Table Label**: テーブルの内容を端的に示す見出し。「主要競合の比較」「拠点別KPI」など。
- **テーブル**: ヘッダー行（列名）＋データ行。列数・行数はコンテンツに応じて自由に決定。推奨上限は列8×行10程度（スライドの可読性を考慮）。
- **意味合い**: 1〜5点。テーブルのデータを根拠に、具体的なビジネス示唆を記述する。抽象的な記述（「検討すべきことが多い」等）は不可。

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: テーブルチャートのドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：テーブルデータと意味合いが明確に入力された場合

1. **Step 1: 内容を確認**し、テーブルのヘッダー・データ行が揃っているか、意味合いが1〜5点あるか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からテーブルチャートを抽出する

### テーブルデータの抽出

議事録から**定量的データ・比較可能な情報**を拾う。以下のような表現を探す：
- 数字・指標・KPI・金額・割合・件数
- 「〜は○○で、△△は□□」のような比較表現
- 複数の項目を横並びで評価している箇所

テーブル設計のポイント:
- **列**: 比較対象（企業名、拠点名、製品名、期間など）をヘッダーに
- **行**: 比較項目（KPI、機能、コストなど）を行に
- **セル**: できるだけ定量値（数字、◎○△×、Yes/No等）で埋める

### 意味合いの抽出・ドラフト

テーブルから読み取れるビジネス示唆を1〜5点で整理する：
- テーブルの数値を根拠にした具体的な示唆にする
- 議事録に示唆が明示されていれば抽出、なければ**戦略コンサルタントとしてドラフト**する
- ドラフトした場合は「※ドラフト」と明記する

---

## Step 2: テーブルチャートのMarkdown出力フォーマット

ユーザーに確認を求める際は、以下のフォーマットで出力する：

```markdown
## テーブルチャート整理結果

**Main Message（※ドラフト）**
〜すべき（最大70文字）

**Chart Title（※ドラフト）**
〜のテーマ（10〜20文字）

**Table Label**
〜の比較 / 〜一覧

### テーブル

| ヘッダー1 | ヘッダー2 | ヘッダー3 | ... |
|---|---|---|---|
| データ1-1 | データ2-1 | データ3-1 | ... |
| データ1-2 | データ2-2 | データ3-2 | ... |

### 意味合い（Implications）
1. テーブルデータを根拠にした示唆1
2. テーブルデータを根拠にした示唆2
3. テーブルデータを根拠にした示唆3（※ドラフト）
```

確認メッセージ例：
> 上記のテーブルチャート整理でよろしいでしょうか？Main MessageとChart Titleも含めて修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートは brand 別に格納されている：

```bash
# stella (16:9, 13.33×7.50 in)
TEMPLATE="<SKILL_DIR>/assets/stellar_aiz/table-chart-template.pptx"

# roleup (A4 横, 11.69×8.27 in, Yu Gothic UI, 茶系)
TEMPLATE="<SKILL_DIR>/assets/roleup/table-chart-template.pptx"
```

`--template` を省略した場合は `--brand` から自動解決される（推奨）。

テンプレートのShape名は実際に確認してからマッピングすること。`references/template-mapping.md` に確認後の対応表を記載する。

### テーブルチャートデータのJSON化

内容を `{{WORK_DIR}}/table_chart_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜のテーマ（10〜20文字）",
  "table_label": "テーブルの見出し",
  "table": {
    "headers": ["ヘッダー1", "ヘッダー2", "ヘッダー3"],
    "rows": [
      ["データ1-1", "データ2-1", "データ3-1"],
      ["データ1-2", "データ2-2", "データ3-2"]
    ]
  },
  "implications": [
    "意味合い1（テーブルの数値を根拠にした具体的な示唆）",
    "意味合い2",
    "意味合い3"
  ]
}
```

**テーブルの仕様:**
- `headers`: 列名の配列。列数に制限なし（推奨上限8列）
- `rows`: データ行の2次元配列。行数に制限なし（推奨上限10行）
- 列数はheadersの要素数で自動決定される
- 列幅はテーブル総幅 ÷ 列数で均等配分される

**意味合いの仕様:**
- 1〜5個の文字列配列
- テーブルのデータを根拠にした具体的なビジネス示唆

### スクリプト実行コマンド

```bash
# stella (default, 16:9)
python <SKILL_DIR>/scripts/fill_table_chart.py \
  --data {{WORK_DIR}}/table_chart_data.json \
  --output {{OUTPUT_DIR}}/TableChart_output.pptx \
  --brand stellar_aiz

# roleup (A4 横, 茶系)
python <SKILL_DIR>/scripts/fill_table_chart.py \
  --data {{WORK_DIR}}/table_chart_data.json \
  --output {{OUTPUT_DIR}}/TableChart_output.pptx \
  --brand roleup
```

`--brand roleup` 指定時は `data.source` が必須（`require_source` でハードフェイル）。

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/TableChart_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] Table Labelがテーブルの内容を端的に示しているか
- [ ] テーブルのヘッダー行と全データ行が正しく反映されているか
- [ ] 意味合いが1〜5点あり、テーブルの数値を根拠にした具体的な示唆になっているか
- [ ] PPTXのmarkitdown出力でプレースホルダーが残っていないか
